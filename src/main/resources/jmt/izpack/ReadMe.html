Installation notes

It is recommended to UNINSTALL any previous version of Java Modelling Tools present on your PC before installing this new release.

What's new in version 0.9.2

JMT

  • Project upgraded to Java 1.7
  • Upgraded installation to izPack 5.0 to support Windows 8.x
  • Default resolution upgraded to 1024x768
  • Code improvements: deep refactoring, added maven support, added unit tests

JMVA

  • Users can now select any station in the model as Reference station (the station used for the computation of the system throughput and the system response time).
  • Code of MoM, CoMoM, and AMVA solution algorithms now part of the stable code branch

JSIM

  • New scheduling algorithms (shortest job first SJF, longest job first LJF, random RAND)
  • Class switch, new more powerful implementation, not available in this release for models with fork/join and finite capacity region
  • Support for using a large number of classes, queues, and finite capacity regions in a single model
  • Several statistics (mean, moments, ...) are generated for the performance indices (response time, number of customers, ...) with the possibility of generating graphics (histogram, density, distribution) on all or a part of the simulated events (results can be saved in CSV files)
  • New simulation control parameter, the simulation can be stopped at a user specified simulation time

Working folder

Since JMT 0.9.0 version the new concept of working folder was introduced.
The working folder is created in the home directory of the user to store the examples and for some working files of JMT. Also the files created by the Loggers (xxx.csv files) are located in this folder. The location of the JMT working folder depends on the operating system of the machine and the user name:
  • on Linux and Unix os in general, the user home is located in path /home/<user name>/JMT
  • on Windows XP the working folder is C:\Documents and Settings\<user name>\JMT
  • on Windows Vista, Windows 7 and later the folder is C:\Users\<user name>\JMT
  • on Mac OS X the folder is /Users/<user name>/JMT
In previous JMT versions, the default working folder was the JMT installation path. If you have stored some JMT models there, we suggest you to move them in the new working folder.
